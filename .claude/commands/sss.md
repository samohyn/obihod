# /sss — save session summary

Сохрани **текущую сессию** Claude Code как curated markdown-файл в [chat/](../../chat/).

## Шаги

1. **Определи имя файла** `chat/YYYY-MM-DD-<topic>.md`:
   - **Дата** — из контекста сессии (`Today's date is …` в system reminder) или `date +%Y-%m-%d`. Если сессия началась вчера, а сейчас сегодня — используй дату старта сессии (см. первое сообщение оператора / `git log` начальных коммитов сессии).
   - **Topic** — kebab-case, 2-4 слова, отражает основной фокус работы. Примеры существующих: `seo-discovery-kickoff`, `panel-backlog-rebuild`, `panel-9pr-sprint`, `panel-nav-home-link-cleanup`. Не «session», не «work», не «misc» — конкретный домен + действие.
   - Если файл с таким именем уже существует — добавь суффикс `-2`, `-3` и т.д.

2. **Собери curated содержимое** (НЕ сырой транскрипт — это явно запрещено в [chat/README.md](../../chat/README.md)). Структура:

   ```markdown
   # YYYY-MM-DD · <Тема сессии>

   **Длительность:** ~N часов · **Роль:** <основная роль / @cpo / @popanel / основная сессия> · **Ветка:** <git branch>

   ## Summary

   5-10 строк прозы: что делали, какой результат, главные развилки.

   ## Ключевые цитаты оператора

   - «...» — дословно, raw, по существу (не парафраз)
   - «...» — 3-7 цитат максимум, только содержательные

   ## Decisions

   - **<решение>** — кем принято (оператор / PO / роль), почему, ссылка на артефакт если есть
   - ...

   ## Созданные / изменённые артефакты

   - [path/to/file.md](../path/to/file.md) — краткое описание
   - ...

   ## Open follow-ups

   - [ ] <что осталось>
   - [ ] <следующий шаг>
   ```

3. **Запиши файл** через Write tool. Используй абсолютный путь `/Users/a36/obikhod/chat/YYYY-MM-DD-<topic>.md`.

4. **Подтверди** оператору одной строкой: имя файла + кол-во цитат/decisions/артефактов. Не дублируй содержимое в чат.

## Что НЕ делать

- НЕ копировать `~/.claude/projects/*/<session>.jsonl` — это сырой транскрипт, запрещён README.
- НЕ сохранять секреты (`.env`, токены, ключи) — даже если они проскакивали в обсуждении.
- НЕ дублировать `.claude/memory/handoff.md` или `specs/<EPIC>/intake.md` — ссылайся.
- НЕ писать filler («Сегодня была продуктивная сессия…») — только содержательные факты.
- НЕ коммитить — папка `chat/` в `.gitignore` (railroad rule #7 в CLAUDE.md).
