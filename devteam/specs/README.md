# devteam/specs/

Артефакты задач. Одна папка на каждую user story: `US-<N>-<slug>/`.

## Структура типовой папки US

```
US-<N>-<slug>/
├── intake.md   # бриф от in (Фаза 1)
├── ba.md       # бизнес-анализ, требования (Фаза 2)
├── sa.md       # системная спека: US, AC, UML, ERD, DoD (Фаза 4)
├── erd.md      # ERD и план миграции (по необходимости, dba/be3)
├── qa.md       # тест-отчёт qa1 или qa2 (Фаза 8)
├── cr.md       # код-ревью от cr (если не в PR, Фаза 9)
└── out.md      # acceptance vs business от out (Фаза 10)
```

Детали — [../WORKFLOW.md](../WORKFLOW.md) §3 «Фазы процесса», §6 «Артефакты и
папки».

Номера US сквозные и монотонно растут, не переиспользуются.
