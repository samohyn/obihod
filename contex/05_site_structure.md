#### 3.2. Long-tail паттерны (по 5+ на вертикаль)

**Арбористика:**
1. «спил {породы} в {районе}» — спил тополя/берёзы/дуба/ели/сосны/яблони × 20 районов = 120 long-tail
2. «удаление аварийного дерева {район} 24 часа»
3. «спил дерева {диаметр} в {районе} цена» (Ø до 20 / 20–40 / 40+ см)
4. «кронирование {породы} {районе}»
5. «удалить пень фрезой {район}»
6. «дерево упало на {объект} {район}» (дом/баню/забор/провода)
7. «арборист с автовышкой {район}»
8. «порубочный билет {район} как оформить»

**Очистка крыш:**
1. «уборка снега с крыши {районе} цена за м²»
2. «сбить сосульки {район} срочно»
3. «очистка кровли УК {район} подряд»
4. «вывоз снега самосвалом {район}»
5. «очистка плоской кровли БЦ {район}»
6. «промальп уборка снега {район}»
7. «штраф ОАТИ за снег на крыше — что делать»

**Вывоз мусора:**
1. «вывоз строительного мусора {район} газель»
2. «контейнер {объём} м³ {район} цена» (8/20/27)
3. «вывоз мусора после ремонта {район}»
4. «вывоз КГМ {район} с грузчиками»
5. «вывоз ТКО {район} по договору с УК»
6. «вывоз старой мебели {район}»
7. «вывоз мусора из СНТ {район}»

**Демонтаж:**
1. «снос дома под ключ {район} цена»
2. «демонтаж дачи с вывозом {район}»
3. «снос {материал} дома {район}» (кирпич/брус/каркас/бетон)
4. «демонтаж бани {район}»
5. «внутренний демонтаж квартиры {район}»
6. «снос пятна под застройку {район}» (B2B)
7. «разбор сарая с вывозом {район}»

**Итого long-tail-паттернов:** ~28 × 20 районов = **~560 уникальных long-tail-комбинаций**, кластеризованных в 80 programmatic-страниц.

### 5. Уникализация programmatic-страниц (правила для service × district)

#### 5.1. Обязательные блоки (без них — `draft` в Payload, страница не публикуется)

| Блок | Что в нём | Метрика контроля |
|---|---|---|
| **mini-case района** | Реальный объект Обихода: фото до/после + дата + ФИО бригадира + 2–3 предложения о специфике | Если поле `relatedCases[]` пусто для этого района — блок «coming soon» + страница помечается `noindex` до появления кейса |
| **2 локальных FAQ** | Уникальные Q&A о районе («Как получить порубочный билет в адм. Раменского ГО?», «Сколько стоит вывоз 20 м³ из СНТ Раменского района?») | Поле `localFAQ[]` Payload, валидация ≥2 |
| **Локальные landmarks** | Минимум 3 КП/мкр/ЖК района упомянуты в тексте | Регулярная проверка по полю `district.landmarks[]` |
| **Локальный регуляторный абзац** | Ссылка на администрацию района + специфика порубочного билета (для арбо) / тарифы ОАТИ (для снега) / Мосгосстройнадзор (для демонтажа) | Поле `district.regulatorNote` |
| **Локальный социальный сигнал** | 2–3 отзыва из Яндекс.Карт по карточке Обихода в районе (через API) | Если карточка <3 отзывов — заглушка «отзывы скоро» (опять `noindex` до набора) |
| **Гео-фото** | ≥1 фото с geo-EXIF из района + schema `contentLocation` | Хук Payload проверяет EXIF при upload |

#### 5.2. Метрика уникальности

**Цель:** ≥60% уникального контента на странице vs template-core.

**Метод проверки (для tech-seo):**
1. Все programmatic-страницы прогоняются через сравнение с эталонным template (только template-блоки: H1-структура, прайс-таблица, общий FAQ).
2. Уникальная часть = все блоки, помеченные в Payload как `local: true` (mini-case, localFAQ, landmarks, regulatorNote, отзывы, geo-фото).
3. Метрика: `(weight_local_blocks / weight_total_page_text) >= 0.6`.
4. Реализация — отдельный CI-чек в GitHub Actions на каждый PR в `service-districts`.

**Запрет на публикацию без miniCase:** Payload-хук `beforeChange` блокирует переход draft → published, если `relatedCases.length < 1` ИЛИ `localFAQ.length < 2` ИЛИ `landmarks.length < 3`.

#### 6.3. Person schema на бригадирах

**Структура (для tech-seo):**

Для каждого бригадира — отдельный объект в Payload-коллекции `persons`:

- Стаж лет
- Сертификаты (массив: Минтруд №782н, ETW, ISA, СРО)
- Фото (квадратное, портретное)
- knowsAbout: массив (арбористика, промальп, демонтаж — соответствующее)

#### 6.4. Кейсы как E-E-A-T-сигнал

`/kejsy/` (с детальными подстраницами):
- Каждый кейс — `CreativeWork` schema + `contentLocation` (район) + `hasPart` (фото до/после) + `VideoObject` (если есть видео на YouTube/RuTube).
- Метаданные: дата, услуга, район, бригадир, длительность, цена за объект, площадь/диаметр/м³.
- 30+ кейсов к месяцу 6 (5 кейсов/месяц с месяца 1).
- Прямая отдача: каждый кейс цитируется в 3–5 programmatic-страницах через `relatedCases[]`.

#### 8.2. Принципы chunking и answer-first

**Каждая programmatic-страница начинается с TL;DR-абзаца (200–300 токенов), который:**
1. Содержит ответ на главный коммерческий вопрос («сколько стоит спил в {районе}»).
2. Цены — с конкретными цифрами и диапазонами («4 900–45 000 ₽»).
3. Бренд + район + услуга — в первых 50 словах.
4. Контактный канал — в первых 100 словах («смета за 10 минут в Telegram/MAX/WhatsApp»).
5. Регуляторный пункт — в первых 200 словах («порубочный билет за наш счёт»).

**Chunking:** каждый H2-раздел = самостоятельный chunk 200–300 токенов. Без «обёртки одного факта в 2000 слов».

### 11. Открытые вопросы / эскалации (стратегические)

#### 11.1. Критические (блокируют запуск SEO-активностей)

1. **[РИСК] ИНН ООО «Обиход» — 7847729123 (СПб, код 78), Я.Бизнес для районов МО.**
    - Я.Бизнес и Я.Карты при создании карточки-филиала по локации в МО требуют подтверждения фактического присутствия в районе (адрес офиса/точки + соответствие региона юр.адресу или ОП).
    - Для полноценного локального ранжирования (локальный пак, «организации рядом») нужна 1 карточка с юр. адресом + N карточек обособленных подразделений (ОП) в МО.
    - **Эскалация → cpo + legal-advisor:** решение по способу — (а) аренда минимальных юр. адресов в 9 районах A, (б) регистрация ОП через ФНС (без открытия юрлиц), (в) перерегистрация ООО на МО, (г) только 1 карточка в Москве + extended geoRadius (слабый локальный пак). **Срок решения: до старта M4 (programmatic-волна 1).**
2. **[РЕСУРС] Кто пишет уникальные FAQ × 20 районов × 4 услуги = 160 локальных Q&A?**
    - Не ChatGPT-генерация без редактуры (фильтр Scaled Content Abuse).
    - **Эскалация → copywriter + research-analyst:** редакционный план — копирайтер пишет с источниками от research-analyst по каждому ГО (адм. сайт, ПП МО, специфика); 20 районов × 8 локальных FAQ (2/услуга) = 160 Q&A; темп 8/неделя за 5 месяцев. Бюджет: оценить с copywriter.
3. **[РЕСУРС] Откуда берём landmarks по 20 районам?**
    - Список КП, мкр, ЖК, СНТ для каждого ГО — нужен ресёрч.
    - **Эскалация → research-analyst:** артефакт `contex/06_districts_data.md` с таблицей district → 5+ landmarks (с источниками: 2ГИС, Я.Карты, Wikipedia, сайт администрации). **Срок: до старта M4.**
4. **[РЕСУРС] Кейсы по районам — кто и когда снимает фото/видео.**
    - Без miniCase публикация programmatic блокируется. Для 9 районов A на старте M4 нужно минимум 9 кейсов (1/район).
    - **Эскалация → cpo + project-manager:** пакет «фото-съёмка с первого объекта» — каждая бригада снимает до/после на телефон + 3 предложения от бригадира. Нужен SOP. Для MVP Раменского — кейс должен быть готов до публикации страницы.

#### 2.1.6. Ключевые hooks

| Хук | Коллекция | Назначение |
|---|---|---|
| `beforeValidate: validateUniqueness` | `service-districts` | Считает Jaccard-similarity `leadParagraph + localFaq` vs template-core по той же `service`. Threshold ≥60% уникального → иначе `ValidationError` |
| `beforeValidate: requireMiniCaseForPublish` | `service-districts` | Если `publishStatus='published'` и `miniCase=null` → `ValidationError('Нельзя публиковать без кейса из района')` |
| `afterChange: revalidateAndPing` | все контентные | `revalidateTag(<page-tag>)` + `revalidateTag('sitemap')` + IndexNow ping при `publishStatus` → `published` |
| `afterChange: anonymizeOldLeads` (cron job) | `leads` | 152-ФЗ: phone/email → hash через 3 года |
| `beforeChange: normalizePhone` | `leads` | E.164 нормализация |
| `beforeChange: ordValidation` | `service-districts` | Если `isOrdMarked=true`, `eridToken` обязателен |

// middleware.ts
import { NextResponse, type NextRequest } from 'next/server'

export function middleware(req: NextRequest) {
  const url = req.nextUrl.clone()
  const host = req.headers.get('host') ?? ''
  const proto = req.headers.get('x-forwarded-proto') ?? 'https'

  // 1. http → https
  if (proto === 'http') {
    url.protocol = 'https:'
    return NextResponse.redirect(url, 301)
  }
  // 2. www → non-www
  if (host.startsWith('www.')) {
    url.host = host.replace(/^www\./, '')
    return NextResponse.redirect(url, 301)
  }
  // 3. Trailing slash policy
  const pathname = url.pathname
  if (!pathname.startsWith('/api/') && !pathname.includes('.') && !pathname.endsWith('/')) {
    url.pathname = pathname + '/'
    return NextResponse.redirect(url, 301)
  }
  // 4. Legacy redirects из коллекции redirects
  const redirect = legacyRedirects.get(pathname)
  if (redirect) {
    if (redirect.statusCode === '410') {
      return new NextResponse(null, { status: 410 })
    }
    url.pathname = redirect.to
    return NextResponse.redirect(url, parseInt(redirect.statusCode))
  }
  return NextResponse.next()
}

#### `beforeValidate` hook — uniqueness и miniCase

```typescript
// payload/hooks/serviceDistricts.ts
import { ValidationError } from 'payload/errors'

const TEMPLATE_TOKENS_CACHE = new Map<string, Set<string>>()

async function getTemplateTokens(req: any, serviceId: string): Promise<Set<string>> {
  if (TEMPLATE_TOKENS_CACHE.has(serviceId)) return TEMPLATE_TOKENS_CACHE.get(serviceId)!
  const sd = await req.payload.find({
    collection: 'service-districts',
    where: { service: { equals: serviceId }, publishStatus: { equals: 'published' } },
    limit: 50,
  })
  // Берём пересечение токенов 5+ опубликованных страниц как «template-core»
  const tokenCounts = new Map<string, number>()
  for (const doc of sd.docs) {
    const tokens = tokenize(richTextToPlain(doc.leadParagraph)
      + ' ' + doc.localFaq.map(f => f.question + ' ' + richTextToPlain(f.answer)).join(' '))
    for (const t of tokens) tokenCounts.set(t, (tokenCounts.get(t) ?? 0) + 1)
  }
  const threshold = Math.max(5, Math.floor(sd.docs.length * 0.6))
  const core = new Set([...tokenCounts.entries()].filter(([, c]) => c >= threshold).map(([t]) => t))
  TEMPLATE_TOKENS_CACHE.set(serviceId, core)
  return core
}

function jaccardUniqueness(docTokens: Set<string>, templateCore: Set<string>): number {
  if (docTokens.size === 0) return 0
  const overlap = [...docTokens].filter(t => templateCore.has(t)).length
  return 1 - overlap / docTokens.size
}

export const validateUniqueness: CollectionBeforeValidateHook = async ({ data, req }) => {
  if (data?.publishStatus !== 'published') return data
  const docText = richTextToPlain(data.leadParagraph)
    + ' ' + (data.localFaq ?? []).map((f: any) => f.question + ' ' + richTextToPlain(f.answer)).join(' ')
  const docTokens = new Set(tokenize(docText))
  const core = await getTemplateTokens(req, data.service)
  const score = jaccardUniqueness(docTokens, core)
  data.uniquenessScore = Math.round(score * 100)
  if (score < 0.6) {
    throw new ValidationError([{
      field: 'leadParagraph',
      message: `Уникальность ${Math.round(score * 100)}% < 60%. Усильте локальный контекст: landmarks, кейс, цены района.`,
    }])
  }
  return data
}

export const requireMiniCaseForPublish: CollectionBeforeValidateHook = async ({ data }) => {
  if (data?.publishStatus === 'published' && !data.miniCase) {
    throw new ValidationError([{
      field: 'miniCase',
      message: 'Нельзя публиковать programmatic без mini-кейса из района (Scaled Content Abuse risk).',
    }])
  }
  if (data?.publishStatus === 'published' && (!data.localFaq || data.localFaq.length < 2)) {
    throw new ValidationError([{
      field: 'localFaq',
      message: 'Минимум 2 локальных FAQ для публикации.',
    }])
  }
  return data
}
```

При неуспехе любого гейта — `ValidationError` блокирует publish, payload показывает редактору конкретную причину.

### 2.10. Открытые вопросы и решения для CTO/оператора

1. **LocalBusiness при ИНН в СПб (ООО «Обиход» 7847729123).** Юрадрес — Санкт-Петербург, услуги оказываются в Москве и МО. Для каждой `district-hub` страницы планируется отдельный `LocalBusiness` через `parentOrganization` ref на главную Org. **Решить:** (а) Регистрируем обособленные подразделения в МО (хотя бы одно — Раменское или Одинцово) и тогда `LocalBusiness.address` = адрес ОП; (б) или ставим юрадрес СПб в Org, а в `LocalBusiness` per district указываем `areaServed: GeoShape` без `address` (валидно по schema.org, но Яндекс.Бизнес для региональности требует точку с адресом). Без решения — Я.Бизнес не зарегистрирует филиалы по 30 районам МО → теряем региональность приоритета №1. См. также Часть 1, раздел 11.1.

2. **Хостинг-провайдер.** CLAUDE.md immutable: Beget. Реальный VPS `155.212.186.204` — IP принадлежит RIPE-блоку без явной идентификации Beget. `whois 155.212.186.204` показывает только `OrgName: RIPE Network Coordination Centre, Country: NL` — это неинформативно (RIPE = регистратор, не хост). Возможно: Selectel/IHC/Timeweb/мелкий провайдер. **Действие:** оператор подтверждает фактического провайдера; если не Beget — обновляем CLAUDE.md immutable (юрисдикция ПДн, S3-провайдер, CDN). Команды для проверки: `whois 155.212.186.204 | grep -i descr`, `nslookup 155.212.186.204`, `dig +short -x 155.212.186.204`.

3. **S3-провайдер для медиа.** Beget S3 (если хостинг Beget), Yandex Object Storage (надёжнее для CDN), Selectel Object Storage. Влияет на: `next/image` custom loader, `images.remotePatterns`, CSP `img-src`, presigned URL генерацию в backend. Решение нужно до начала фазы 1 (дизайн-система + первая посадочная).

4. **CDN.** Beget CDN, Yandex Cloud CDN, Selectel CDN, Cloudflare (нюанс: западная юрисдикция, риски с 152-ФЗ при кешировании страниц с ПДн — допустимо только для статики). Для CWV критично: CDN-узлы в МО + СПб. Влияет на TTFB <400 мс target.

5. **EXIF geo-метаданные фото.** Backend-developer на upload **скрабит EXIF** для безопасности. Но для `ImageObject.contentLocation` в schema нам нужны geo-координаты. **Решение:** перед скрабом извлекать EXIF GPS в `media.geoLocation` (Point поле в коллекции `media`), затем удалять EXIF из бинарного файла. Координаты живут в БД, фото публичны без EXIF.

6. **Trailing slash.** Зафиксирован slash на всех маршрутах (см. middleware). Часть 1 раздел 1 «Дубли» пишет «trailing slash стандарт Next.js» — Next 16 default = без слеша, поэтому переопределяем через middleware. Убедиться, что эту политику все знают: backend-developer (canonical URLs в API responses), copywriter (внутренние ссылки в richText блогов).

7. **«Региональность» в Я.Вебмастере без обособленных подразделений.** Если решение по п.1 = «без ОП», региональность по 30 районам МО придётся получать через альтернативные механизмы: качественный локальный контент + Я.Бизнес-карточки (требуют физический адрес для верификации) + добавление района как preset в Я.Вебмастере. Это slower и менее надёжно. Эскалация к seo-expert.

8. **Что блокирует tech-seo в работе backend-developer:**
   - Контракт `POST /api/revalidate?tag=<tag>` с `x-revalidate-secret` header.
   - `payload generate:types` → `lib/payload-types.ts` для типизации SEO-фабрик.
   - Endpoint `/api/indexnow-ping` (или прямой вызов из `afterChange` hook).
   - EXIF-extraction до скраба в media-upload (см. п.5).
   - Возврат полного `{service, district, sd}` объекта при server-side fetch для programmatic page (depth: 2 в Payload query).

9. **Что блокирует tech-seo в работе frontend-developer:**
   - `<JsonLd>` Server Component готов в `components/seo/JsonLd.tsx`.
   - `<Breadcrumbs>` с интегрированным BreadcrumbList JSON-LD.
   - `<RelatedLinks>` с детерминированной выборкой sibling/cross-service/cross-district.
   - Запрет на native `<img>` вне `next/image` — линтер-правило `@next/next/no-img-element`.
   - Запрет на синхронные сторонние скрипты — линтер `react/no-script-url`.
   - Использование `next/font/local` с правильным subset.

10. **OG-картинка — fallback.** Динамическая `opengraph-image.tsx` для programmatic; для блога/кейсов — статичная из коллекции (`ogImage` поле). Нужно подтвердить, что ImageResponse в Next 16 на Beget/выбранном VPS работает без edge runtime (используем `runtime = 'nodejs'` явно).
