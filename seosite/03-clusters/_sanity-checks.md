## Q-2 — `chistka-krysh` или `ochistka-krysh`?

См. начало [chistka-krysh.md](chistka-krysh.md).

**Сигнал по wsfreq (Москва+МО):**
- «чистка крыш/кровл» — 36 уникальных ключей, sum freq = **888**
- «очистка крыш/кровл» — 28 уникальных ключей, sum freq = **0** _(в выборке wsfreq не оказалось)_

**Сигнал по count подсказок Yandex (sug_par + Keys.so):**
- «чистка» лидирует 36 vs 28 (1.29x)

**Рекомендация re:** **«чистка крыш»** — основной канон (slug `chistka-krysh`).
Текущий канон в БД (`ochistka-krysh`) **мигрировать через Redirects**. Title/H1/мета — «чистка крыш».
Финальный ADR — за seo1 после полного wsfreq (когда JM устаканится).

## Спрос по 8 пилотным районам

Wsfreq собрана только для топ-200 приоритетных ключей (JM-сервис нестабилен в окне сессии); районные long-tail модификации в основном остались с freq=0. Финальный отбор — после полного wsfreq.

| Район | KW в pool | Sum wsfreq (МСК+МО) | Замечание |
|---|---:|---:|---|
| Одинцово | 14 | 644 | сильный сигнал, top — vyvoz-musora |
| Красногорск | 6 | 0 | нужен дозабор wsfreq |
| Мытищи | 5 | 0 | нужен дозабор wsfreq |
| Химки | 4 | 0 | нужен дозабор wsfreq |
| Истра | 1 | 0 | малое покрытие в pool, **слабый сигнал** |
| Пушкино | 3 | 0 | нужен дозабор wsfreq |
| Раменское | 3 | 0 | нужен дозабор wsfreq |
| Жуковский | 0 | 0 | **очень слабый сигнал** — не оказалось в pool |

**Решение для seo1 / `po`:**
- Одинцово, Красногорск, Мытищи, Химки, Пушкино, Раменское — оставить в Wave 1 (есть pool-сигнал).
- **Истра** — пограничный случай: 1 ключ в pool. Не удалять из tree, но в US-6 контент-плане понизить приоритет (ниже остальных пилотных).
- **Жуковский** — пограничный случай: 0 ключей в pool. Это связано с тем, что seed не содержал прямого «жуковский»-маркера, а sug_par расширил недостаточно. **Рекомендация:** оставить в tree (legend-обещание оператора), но в Wave 2.5 запросить в JM `sug_par` дополнительно по seed «спил деревьев жуковский / вывоз мусора жуковский / чистка крыш жуковский / демонтаж жуковский» для дополнительного покрытия.
