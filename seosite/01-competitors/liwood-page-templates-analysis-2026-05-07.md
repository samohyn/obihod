### 3.4 T4 — Programmatic Service×City (`/services/udalenie-derevev/himki/`) ⭐ КЛЮЧЕВОЙ ШАБЛОН

**Это самый важный шаблон для нашего эпика SEO-COMPETE-3 US-3..US-9.**

- **Title:** **отсутствует** (критическая ошибка — это объясняет, почему liwood держит лишь 155 индекса в Keys.so)
- **H1:** «Удаление деревьев в Химках» (только город в названии)
- **Meta description:** нет
- **Block sequence (~1500-1800 слов, 11 блоков):**
  1. Breadcrumbs (Подольск пока без crumb-уровня города в URL-структуре, схемой не размечен)
  2. Hero (H1 с городом + 2-3 предложения вступления + 2 CTA)
  3. «Какие деревья и как следует удалять» (методы валки, generic)
  4. **«Выгодное предложение» — скидка 15%** (кросс-промо)
  5. Таблица цен (идентична T2)
  6. Калькулятор (идентичен T2)
  7. Форма обратного звонка
  8. «Наши преимущества» (6 пиктограмм, generic)
  9. Фотогалерея (generic — не из Химок)
  10. «Как формируется стоимость»
  11. **City-specific блок «Мы оказываем услуги во всех городах и ГП Химкинского района»** — список 30+ микротопонимов (Алешкино, Барашки, Бутаково, Вашутино, Поведники, …)
  12. Повторная форма
- **Что уникально для города (~8-12% контента):** только H1, hero-вступление (2-3 предложения), и блок-список микрорайонов в подвале страницы. По проверке 3 городов (Химки, Подольск, Мытищи) — **подход одинаков везде: token-replace + список населённых пунктов района**.
- **Что НЕ city-specific:** Title (если есть), фото, отзывы, цены, адрес офиса (везде Москва, Старокачаловская д.3), нет упоминаний транспорта/метро/локальных особенностей.
- **CTA:** агрессивно — 7+ точек конверсии (2 формы, телефон ×2, WhatsApp, кнопки в калькуляторе).
- **Trust:** generic — преимущества и форма.
- **Внутренняя перелинковка:** crumb на T2, на 4 услуги (обрезка плодовых, опрыскивание, порубочный билет, удаление пней). **Нет sibling-перелинковки между городами на самой city-странице** — только в mega-menu.
- **SEO — критические упущения liwood:**
  - Title часто не задан или не содержит город
  - JSON-LD `LocalBusiness` / `Service` с `areaServed` отсутствует везде
  - Breadcrumbs Schema нет
  - Нет фото из города, нет локальных отзывов
  - Адрес в Москве, не в городе services
  - **HTTP 404 на `/balashikha/`** при том что он в mega-menu — техдолг
- **Реальный охват:** 49 городов МО × 1 услуга = **только под удаление деревьев**. Под обрезку, опрыскивание, альпинизм — городских страниц **нет**. Это огромная неиспользованная territory liwood, и наш шанс.

**Оценка SEO-зрелости T4 у liwood: 4/10.** Они держат 49 city-URL только за счёт того, что под удаление деревьев конкуренция в SERP слабая. Делая SD-шаблон с правильным Title+Meta+Schema+уникальными hero-абзацами, мы их обгоняем по позициям при равном объёме URL.

## 5. Уникальные приёмы liwood

1. **Title-формула с телефоном** на главной — `«Бренд — категория в Москве | +7…»` повышает CTR в SERP. Не везде (на T2/T3 уже без телефона), но на T1 это сильно.
2. **Mega-menu как city-distributor** — в выпадающем меню под услугой «Удаление» — все 49 городов. Это и навигация, и SEO-перелинковка через все страницы сайта.
3. **«Схема работы» с нумерацией 6 этапов** на главной — наглядный trust-блок.
4. **«10 причин удалить дерево» + «самостоятельно vs специалист»** на T2 — content-marketing внутри коммерческой страницы, добавляет 800+ слов LSI-контента.
5. **Интерактивная Я.Карта объектов** на главной — visual social proof.
6. **Калькулятор-сметка как воронка** — 4 услуги через единый калькулятор.
7. **Список микрорайонов района** в подвале city-страницы (30+ топонимов на каждой) — несмотря на token-replace подход, это даёт hyper-local long-tail охват («удаление деревьев Алешкино», «… Бутаково»).

## 6. Recommendations для шаблонов Обиход (newui/)

Ниже план first-wave / second-wave / not-needed. Для каждого шаблона — обоснование на основе того, что у liwood работает или, наоборот, провисает (и потому это наша возможность).

### Second wave — 4 шаблона (после релиза первой волны, эпик SEO-COMPETE-4 / USS-12)

| # | Шаблон | Обоснование |
|---|---|---|
| **W2.1** | **T3 Под-услуга (тип работы)** | Усиляет MOFU-интент («спил деревьев цена», «вывоз снега крыша цена»). У liwood это 25 страниц с 1800-2000 слов — мы можем сделать так же или лучше. **Не критично для первой волны**: сначала идут 4 pillar T2, потом наращиваем под-услуги. |
| **W2.2** | **T9 Блог-индекс + Статья (с E-E-A-T fix)** | У liwood статьи 400-450 слов без автора и Schema. Наша возможность — Article + Author Schema, ToC, длиннее 1500+ слов, ссылки на авторов из `/team/`. Это US-11 (4 random RU authors уже seed'нуто в текущей ветке). Шаблон закладывает long-term content-marketing. |
| **W2.3** | **T10 Отзывы + AggregateRating Schema + сбор отзывов** | У liwood отзывы — самый слабый блок (текст без фото/звёзд/Schema). US-9 уже Done в backlog по отзывам — нужен dedicated шаблон с карточкой, фильтрами по услугам/районам, AggregateRating, скринами Я.Карт, формой «оставить отзыв». **Сильный CR-эффект.** |
| **W2.4** | **T8 Кейс / Портфолио (`/cases/[slug]/`)** | Liwood ограничился generic-галереей без кейс-стади. Мы строим полноценный кейс-шаблон: фото до-после + клиент (имя/тип объекта/УК/частный) + район + задача + сроки + цена + цитата клиента + видео. Каждый кейс — отдельный URL для long-tail и B2B-сигнала. Идёт после W1 потому что нужен реальный контент-фотобанк. |

### Зона исключения — это рендерим page-by-page в site/ напрямую без шаблона newui/

- **T7 Прайс-лист** — таблица + текст, не нуждается в шаблоне (страница-одиночка). Brand-guide §6 покрывает.
- **T5 Промо/Акция** — у liwood всего 3 промо. Делаем 1-2 страницы как landing, **landing-шаблон уже есть** (`lp-site` команда, brand-guide §1-14). Не отдельный шаблон.
- **T12 Вакансии** — не SEO-priority.
- **/sitemap/, /politika-…/, /how-to-order/, /faq/** — в Wave 1 покрыты T11-shell.
- **B2B-страницы (`/dlya-uk/`, `/dlya-zastroyshchikov/`)** — дублируют T2 + специальный hero+форма. Делаем page-by-page на base layout, не отдельным шаблоном.

### Сводная программа

```
Wave 1 (US-3..US-9 + US-11): T1 + T2 + T3 (только под Pillar1 для запуска) + T4 + T6 + T11-shell = 5 шаблонов
Wave 2 (US-12 / далее): T3 (полноценная) + T9 + T10 + T8 = 4 шаблона
Не делать как шаблон: T5, T7, T12, B2B-LP (page-by-page на base)
```

**Критическое для эпика SEO-COMPETE-3 US-3..US-9:** без W1.1 (T4 Service×District) эпик не закрывается. Это первый шаблон, который должен попасть в `newui/`. Условия его конкурентности vs liwood — уникальные hero-абзацы (минимум 150 слов на каждый город, не token-replace), Title+Meta+Schema (3 пункта где liwood проваливается), фото из района (наш фотобанк), JSON-LD `LocalBusiness` с `areaServed: { "@type": "City", "name": "<district>" }`. С этими 4 настройками мы при равном объёме URL обгоняем liwood по позициям в SERP.
