## Дробление подуслуг

- **3 уровня:** /services/[pillar]/[sub-service]/[district]/
- Глубина по обрезке — по 20+ пород деревьев (яблоня/груша/слива/абрикос/...)
- Programmatic для удаления деревьев = service × 40 районов = 40 уникальных URL только в одном кластере

## Гео-страницы

- **40 районов МО** только в кластере «удаление деревьев»
- Остальные кластеры (обрезка, лечение, промальп) — без programmatic district-карты
- Нет /msk/ root-сегмента — гео живёт внутри service-кластера

## B2B

- **Отдельного раздела нет** (явный пробел в структуре)
- Нет /b2b/, /uk/, /korporativnyim/
- /services/promyshlennyy-alpinizm/ implicitly B2B

## Блог

- Есть /info/articles/ — **85 URL** (значительный объём)
- Темы: обрезка по сезонам и породам, болезни плодовых, регуляторика (порубочный билет, штрафы), уборка снега, лечение, ландшафт
- Темы дублируют pillar-структуру → сильная внутренняя перелинковка

## URL-паттерны

1. `/services/[pillar]/` — pillar
2. `/services/[pillar]/[sub-service]/` — подуслуга
3. `/services/[pillar]/[district]/` — programmatic гео (только udalenie-derevev)
4. `/info/articles/[topic]/` — статьи
5. `/gallery/[topic]/` — галерея

**Канонический паттерн:** `liwood.ru/services/[pillar]/[sub-or-district]/`. Чистая 2-3-уровневая иерархия — это и есть «эталон по SEO» из контекста оператора.
