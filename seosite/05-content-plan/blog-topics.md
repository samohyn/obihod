## Принципы отбора

1. **Каждой теме — кластер из neuro-info.md.** Без target-ключа в темник не
   попадает.
2. **3 pillar внутри блога:**
   - **Pillar A — комплекс / Обиход-method:** статьи под брендовые и общие
     запросы про услугу 4-в-1 (С1, C14, C15) → внутренние ссылки на
     pillar-услуги и `/foto-smeta/`.
   - **Pillar B — регуляторика и право:** разрешения, штрафы, СНиП, нормы
     работы на высоте (C2, C4, C6, C17–C19, C5, C9, C11, C20). Сильный
     B2B-крючок, питает `/b2b/shtrafy-gzhi-oati/`.
   - **Pillar C — сезонные и хозяйственные практики:** обрезка
     плодовых/хвойных, сортировка, утилизация (C3, C7, C8, C10, C12, C13,
     C16, C22). B2C-аудитория (дачники, владельцы загородных домов).
3. **Long-tail из C2 (47 ключей про спил на своём участке)** разбиваем на
   3–4 статьи под разные углы: «закон», «штраф», «по породам» (берёза, ель,
   сосна), «летом / в сезон».
4. **Нейро-формат обязателен:** TL;DR в начале, списки, таблицы, FAQ-блок —
   чтобы Я.Нейро / Perplexity цитировали.
5. **Не пишем общие статьи без target-ключа.** Если для темы нет кластера в
   neuro-info — выкидываем (C21 «содержание в Word» — мусорный кластер,
   игнорируем).
6. **Не повторяем конкурентов 1:1.** E-E-A-T: фото объекта, цифры
   («12 800 ₽ за объект», не «от 1 000 ₽»), автор из `/avtory/<slug>/`, <!-- obihod:ok -->
   ссылка на кейс.
7. **Анти-TOV запрещён** (см. Vocabulary Don't): «услуги населению», <!-- obihod:ok -->
   «индивидуальный подход», «в кратчайшие сроки», «от 1 000 ₽», «имеем <!-- obihod:ok -->
   честь».

## Что НЕ делаем

- Не пишем статьи без target-ключа из neuro-info.md.
- Не пишем по C21 (содержание в Word — мусорный кластер из локального
  fallback-кластеризатора).
- Не дублируем темы конкурентов 1:1: liwood закрывает «как выбрать
  арбориста» — мы заходим через метод «Обиход» и кейсы, не через общие
  советы.
- Не уходим в анти-TOV: «услуги населению», «индивидуальный подход», <!-- obihod:ok -->
  «в кратчайшие сроки», «от 1 000 ₽», «имеем честь предложить». <!-- obihod:ok -->
- Не зашиваем точные даты публикации — это распределяет `po`.
- Не пишем meta title/description — это `seo2` после согласования темника.

## Структура каждой статьи (для Wave 1 production)

1. **TL;DR блок** в начале — 2–3 предложения с прямым ответом на
   target-ключ (нейро-цитирование Я.Нейро / Алиса / Perplexity).
2. **H2-структура из ≥4 секций** с ключами из кластера в подзаголовках.
3. **Списки и таблицы** обязательны (нейро-парсинг).
4. **FAQ-блок 3–5 вопросов** в конце (target FAQPage schema от `seo2`).
5. **Внутренние ссылки на pillar/sub-service** — 3–5 на статью, анкорный
   текст по target-ключам соседних страниц.
6. **Автор** из `/avtory/<slug>/` (после US-3 / OBI-3) — арборист,
   бригадир или эксперт по B2B-договорам.
7. **Рекомендованные кейсы** — 1–2 ссылки на `/kejsy/<slug>/`, если есть
   релевантный объект в портфолио.
8. **CTA в конце** — ссылка на `/foto-smeta/` («Смета за 10 минут — пришлите
   фото в Telegram / MAX / WhatsApp») или на профильный калькулятор.
9. **Объём:** ≥ 800 слов, в среднем 1 200–1 800 (под топ-10 Яндекса).

## Hand-off

- → **`po`** — распределить даты публикации (10/мес × 3 месяца), назначить
  авторов из команды, поставить в Linear под US-6.
- → **`seo2`** — meta title / description / Open Graph по каждому slug,
  Article + FAQPage + Author schema.org разметка, breadcrumbs.
- → **`lp`** — конвертирующий блок в каждой статье: «Смета за 10 минут по
  фото» с ссылкой на `/foto-smeta/` + CTA в Telegram / MAX / WhatsApp.
- → **`art`** — иллюстрации и фото объектов (после US-6 fal.ai pipeline,
  стиль документальный, без стоков, см. бренд-визуал в
  [contex/03_brand_naming.md](../../contex/03_brand_naming.md)).
- → **`fe1` / `fe2`** — рендер блок-схемы в Payload коллекции `Blog`
  (TL;DR-блок, FAQ-аккордеон, блок «связанные услуги», блок «связанные
  кейсы»).
- → **`qa1`** — проверка по чек-листу: TL;DR есть, FAQ есть, ≥ 3 внутренних
  ссылок на pillar, target-ключ в H1+H2+первом абзаце, нет анти-TOV
  словаря.

## Что блокирует Wave 1 production

1. **Wordstat дозабор частот по C2 (47 ключей)** — нужен `re` через
   `seosite/scripts/jm_wsfreq_micro.py` (50-kw батчи, ~1 ₽/батч),
   чтобы расставить точный приоритет внутри Pillar B.
2. **`/avtory/<slug>/` коллекция в Payload** — US-3 (`po` ставит в Linear
   как зависимость US-6).
3. **`/foto-smeta/` посадочная страница** — нужна для анкорных ссылок из
   статей #1, #11, #21. Пилот US-? (по sitemap-tree v0.2 это новый pillar
   v0.2). Без неё ссылки в статьях сейчас уйдут в `/raschet-stoimosti/`
   как fallback.
4. **Решение оператора по Q-2 (`ochistka-krysh` vs `chistka-krysh`)** не
   блокирует темник Wave 1 — крыши покрываются в Wave 2 темника (M4–M6),
   к тому моменту слаг будет финализирован.
