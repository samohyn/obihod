# Photography · foundation

**Статус:** v1.0 · approved 2026-04-24
**Зона:** art-директорские правила съёмки кейсов, коммерческой фотографии, OG-обложек.

## Принцип

**Документальная съёмка реальных бригад на реальных объектах.** Никаких stock-шаблонов «рукавицы + улыбка», никаких catalog-shot на белом. Обиход — не e-commerce и не премиум-студия. Визуальный язык — репортаж дворника-бригадира, а не глянцевый кейс Ростелекома.

## Виды кадров

| Тип | Цель | Когда |
|---|---|---|
| **Before/After** | Доказательство результата | Каждый выезд, обязательно парой |
| **Процесс** | Техника, компетенция, безопасность | Репортажные кадры в момент работы |
| **Бригада** | Лицо компании, человеческий масштаб | Опциональные портретные — для «Команда» |
| **Техника** | Автовышка, альпинисты, измельчители | Деталь-кадры (крупный план, не макет) |
| **Обложка** | Hero / OG / блог | Постановочные кадры из реальных выездов |

## Правила кадрирования

- **Соотношение сторон:**
  - Before/After — `4:3` (стандарт телефонной камеры, легко снимать бригадирам)
  - Hero — `16:9` или `21:9` (cinematic)
  - OG — `1200×630` (`1.91:1`) — стандарт Facebook/Twitter/VK
  - Quadratic — `1:1` для Telegram/MAX/Wazzup превью
- **Rule of thirds:** объект в точках третей, не в центре.
- **Контекст обязателен:** объект + часть двора / дома / местности. Голый объект в пустоте не работает — пользователь не поймёт масштаб.
- **Порядок до/после:** снимаем с **одной точки** — штатив желателен, но не обязателен. На телефоне бригадир делает первый кадр, возвращается в ту же точку после работы.

## Свет

- **Природный дневной свет** — предпочтителен. Overcast (пасмурно) = идеал для детализации, нет пересветов.
- **Утренний / вечерний золотой час** — для hero-обложек лендингов услуг.
- **Искусственный свет:** только для зимних / вечерних кейсов. Без цветных фильтров.
- **Без вспышки в упор** — плоские пересвеченные фото не ложатся в бренд.

## Цвет и пост-обработка

- **Базовая коррекция:** контраст, экспозиция, тени/света — стандарт.
- **Теплота:** +100-200 K к естественной для «уютного» ощущения (Caregiver).
- **Насыщенность:** +5-10%, не больше. Перенасыщенные цвета = Instagram-filter, ломает бренд.
- **Градиенты/виньетка:** запрещены.
- **Filter presets:** если используем — только кастомный LUT согласованный с `art`. Lightroom VSCO / default instagram — запрещены.

## Что в кадре должно быть

✅ Объект работы (дерево, крыша, куча мусора, сарай)
✅ Часть территории / контекст двора
✅ Маркер бренда (спецодежда бригадира с логотипом, бейдж на машине) — опционально
✅ Инструмент в работе (бензопила, альпснаряжение, автовышка) — для «процесса»
✅ Следы работы (опилки, подстрижка, чистая крыша, убранная площадка) — для «после»

## Что в кадре НЕ должно быть

❌ **Лица клиентов** без подписанного согласия (152-ФЗ). Размываем / закрываем / кадрируем.
❌ **Номера машин клиентов** — читаемые госномера из кадра убираем.
❌ **Документы с ПДн** — договоры, паспорта, счета.
❌ **Стороннее имущество** в фокусе (соседний участок, чужие постройки).
❌ **Бренды конкурентов / подрядчиков** (техника взятая в аренду — лого замазываем).
❌ **Бардак за кадром** — если снимаем рабочую зону, она должна быть в рабочем состоянии, не в хаосе.
❌ **Сигареты / алкоголь / нецензурные надписи** — очевидно.
❌ **Stock-models** в спецодежде — только реальные бригады.

## Специфика услуг

### Арбористика
- **Before:** аварийное дерево / сухостой / наклонённое над крышей — опасность видна.
- **Процесс:** альпинист на страховке в кроне, спил крупных веток контролируемый. Шлем + индивидуальная страховка видна (безопасность = бренд).
- **After:** пень или пустое место, убранные ветки, чистый газон. Щепа (chipper) — опционально как акцент.

### Крыши
- **Before:** наслоение снега, сосульки, забитый водосток.
- **Процесс:** работник на верёвке с лопатой, крыша на половину чистая.
- **After:** чистая крыша, видимое небо/солнце.

### Мусор / демонтаж
- **Before:** куча мусора / старый сарай / бардак.
- **Процесс:** контейнер + загрузка, или демонтаж в процессе с видимыми остатками.
- **After:** ровная площадка, готовая к новому использованию.

## Формат хранения

- **Исходники:** `assets/photos/<YYYY-MM-DD>-<district>-<cluster>/` (HEIC / JPG / RAW / PSD). Не коммитятся в git (см. `assets/README.md`).
- **Prod-копии:** `site/public/cases/<slug>/{before,after,process}.jpg` — уже обработанные, оптимизированные. Next.js Image автоматически делает WebP/AVIF.
- **Метаданные:** EXIF сохраняем (дата + геолокация опционально для programmatic SEO — «кейс в Раменском снят 24 апреля»).

## Анти-паттерны конкретно для Обихода

- ❌ «Счастливый работник с поднятой рукой в жёлтой каске» — stock-клише.
- ❌ Wide-angle fish-eye эффекты — вызывают доверие меньше чем ровная перспектива.
- ❌ B&W фотография — мы работаем с живыми природными цветами, ч/б теряет суть.
- ❌ Золото-серебро «премиум» LUT — не наш сегмент.
- ❌ Instagram reels-vertical кадрирование без desktop-версии — для сайта важен `4:3` / `16:9`.

## Процесс

1. Бригадир делает кадры на телефон **до начала работы** (пока не забыл) и **после** — с одной точки.
2. Отправляет оператору в Telegram канал `#photos-raw` (TODO: завести).
3. `art` / `cw` раз в неделю разбирают: отбор, обработка, публикация в кейсы.
4. Отобранные кадры → `assets/photos/` (исходники) + `site/public/cases/` (прод).
5. Для LP района — OG-image собирается из подходящего кейса + wordmark overlay (TODO: OG endpoint).
