ОБИХОД

Комплексный подрядчик 4-в-1 для Москвы и МО: арбористика, чистка крыш, вывоз мусора, демонтаж. Архетип Caregiver + Ruler. Этот документ — единая точка входа для дизайнера, копирайтера, фронтендера и любого нового агента команды.

Хозяйство в надёжных руках
01 · Identity

Кто мы и чего не говорим

Выжимка из бренд-стратегии. Детали, scorecard финалистов и обоснование выбора — в 03_brand_naming.md.

Позиционирование

Для владельцев загородных домов и управляющих компаний в Москве и Подмосковье, которые устали от фрагментированного поиска подрядчиков и смет «от X ₽», Обиход — комплексный сервис 4-в-1 по фиксированной цене за объект, со сметой за 10 минут в мессенджере и юридической ответственностью за штрафы ГЖИ/ОАТИ.

Архетип

Caregiver (70%) — забота о хозяйстве, защита от проблем, «мы присмотрим».
Ruler (30%) — порядок, контроль, компетентность.

Отвергнуто: Outlaw (Дровосек/Лесорубы), Jester (Грузовичкоф), Explorer (Liwood).

Essence

Порядок под ключ — операционное кредо.
Хозяйство в надёжных руках — эмоциональное обещание.

Оффер (immutable)

Фикс-цена за объект · смета за 10 минут по фото · абонемент «зима-лето» · штрафы ГЖИ/ОАТИ на подрядчике.

Расхождение с бренд-брифом

Визуальный бриф в 03_brand_naming.md описывает палитру как «тёмный графит + тёплый песок + приглушённый терракот / густая охра». Фактическая палитра в globals.css — зелёный + янтарный + кремовый. Это решение имплементации; бриф остаётся как исторический контекст. Источник истины — код. При ребрендинге пересматриваются оба артефакта синхронно.

04 · Контраст

WCAG 2.2 — что на что кладём

Приёмка цветовых пар по WCAG 2.1/2.2: AA normal text = 4.5:1, AA large text (18px+ bold / 24px+) = 3:1, AAA normal = 7:1.

ПараПримерRatioNormalLargeГде применяется
ink на bg Обиход 15.1 : 1AAAAAA Весь body-текст, заголовки
on-primary на primary Позвонить 8.9 : 1AAAAAA Primary-кнопки, бейджи
primary на bg Обиход 7.1 : 1AAAAAA Ссылки, иконки, eyebrow
ink-soft на bg-alt строка таблицы 13.8 : 1AAAAAA Чередование таблиц
muted на bg подпись 4.1 : 1AA✕AA Только large / caption; избегать body
accent-ink на bg Черновик 4.7 : 1AAAAA Статусы, warning-бейджи
accent на bg не для текста 2.3 : 1AA✕AA✕ Только фоновые акценты, иконки 24+
error на bg Ошибка 5.4 : 1AAAAA Сообщения валидации, delete

Фокус-outline — всегда accent 2px с offset 2px (см. .btn:focus-visible в globals.css). Этот приём безопасен, потому что outline идёт по большой площади кнопки, а не является текстом.

Что видит пользователь после действия

Формы сбора заявок — главный конверсионный механизм. После отправки пользователь должен точно знать что заявка ушла, куда ждать ответ и что делать если что-то не так. Полный spec — feedback-states.spec.md, копи — microcopy.md.

Полный 5-стейтовый flow отправки заявки

  1. Idle — форма в дефолте, submit disabled (нет телефона/фото)
  2. Filled — все поля валидны, submit активирован
  3. Submitting — submit button «Отправляем…» со spinner, форма pointer-events: none, никаких модалок
  4. Success — форма целиком заменяется на success-card с ID заявки (см. выше)
  5. Error — форма НЕ сбрасывается, error toast сверху, submit снова активен, retry

Anti-patterns

«Спасибо!» без контекста

Что произошло? Что дальше? Пользователь не должен гадать.

Модалка на весь экран после submit

Блокирует UX без причины. Inline success-card на месте формы.

«Упс!» «Ой!»

Инфантилизация. Обиход = matter-of-fact.

Toast без action

Ошибка без «что сделать» = frustration. Всегда либо link-action, либо retry.

Error только цветом

WCAG 1.4.1. Иконка + текст + цвет — все три одновременно.

Автодисмисс для ошибок

Success можно dismiss через 3с, error — только ручным закрытием пользователем.

11.5 · Страницы ошибок

Когда что-то пошло не так

Error pages — это микро-доверие. У конкурентов на WP/Bitrix чаще всего «Apache 503 Service Unavailable» во весь экран на дефолтном шрифте — это «у них всё развалилось». У Обихода — matter-of-fact страница с понятным кодом, человеческим объяснением, путём наружу и контактом, который точно работает (телефон / Telegram). Архетип Caregiver: «упало — поднимем, вы пока сюда».

Принципы копирайта (TOV-черновик от art, финал — cw)

Никогда «Упс!» / «Ой!»

Инфантильно. У Обихода ошибки звучат как «бригадир спокойно описывает проблему»: «Сервер не отозвался», «Технический перерыв».

Всегда CTA-выход

Минимум одна кнопка-действие. Запасной канал (Telegram / телефон) — как fallback под ней.

Объяснить, не извиниться

Caregiver = «вижу проблему, делаю». «Бывает после деплоя — возвращается за минуту» = доверие. «Простите, у нас сложности» = слабость.

Код виден, но не доминирует

«HTTP 502 · BAD GATEWAY» — eyebrow-стиль uppercase mono 13px, не h1. Технику оставляем для тех кто читает, у остальных — заголовок-человек.

Иллюстрация = метафора, не картинка

Компас (404), пила-чихалка (500), оборванная цепь (502), табличка-перерыв (503), антенна без сигнала (offline). Всё в lin-art currentColor 96×96. Никаких photorealistic AI / стоковых.

Sentry / log — обязательно

500 / 502 / 503 шлют событие в Sentry с контекстом (URL, user-agent, request-id). На странице — невидимо, но в коде app/error.tsx вызов captureException.

Spec для реализации — в components/error-state/error-state.spec.md (создаётся ui по art-brief). Тексты — финализирует cw под TOV. Развёртывание 502/503 на nginx — задача do (Beget reverse-proxy конфиг).

Как звучит Обиход

Анти-TOV слова блокируются хуком .claude/hooks/protect-immutable.sh — если попытаться закоммитить «в кратчайшие сроки» в site/, хук упадёт.

Шкалы (Nielsen Norman)

Funny◄────────────●─────►SeriousСерьёзный, но не казённый
Formal◄──────●───────────►CasualДеловой с тёплыми вкраплениями
Respectful◄●──────────────────►IrreverentМаксимальное уважение
Enthusiastic◄────────────●──────►Matter-of-factДелом, без восклицаний

Пять принципов

  1. Слово держим, дело делаем. Не обещаем того, что не сможем. Сказали «смета за 10 минут» — значит 10.
  2. Цифры и срок — всегда конкретные. Не «от 1 000 ₽», а «12 800 ₽ за объект». Не «в течение недели», а «послезавтра до 12:00».
  3. Техникой не пугаем. СРО, класс отходов, ПОР — переводим на «что это значит для вас».
  4. Не обесцениваем и не задираем. Мы не «дешевле всех» и не «премиум». Точно по делу, по рынку, с гарантией.
  5. Ответственность — это наша работа. Штрафы ГЖИ и ОАТИ берём не как бонус, а как норму.

Словарь

Да

  • «обиход», «хозяйство», «объект», «порядок»
  • «сделаем», «приедем», «закроем вопрос»
  • «12 800 ₽ за объект»
  • «послезавтра до 12:00»
  • «берём штрафы на себя», «отвечаем по договору»
  • «снег / дерево / мусор / снос»
  • «СРО 1 млрд, страховка 10 млн, ГЛОНАСС»

Нет

  • «услуги населению», «клиент» (в B2C)
  • «имеем честь предложить», «осуществляем деятельность»
  • «от 1 000 ₽»
  • «в кратчайшие сроки»
  • «гарантируем качество» (пустое), «индивидуальный подход»
  • «твёрдые коммунальные отходы», «древесно-кустарниковая растительность» (в B2C)
  • «мы дорожим репутацией»

Хипстерская ассоциация, ломает корпоративную приёмку в УК/ТСЖ.

Q 01Сколько стоит спил дерева 12 м?

Базовая цена — от 9 500 ₽ за дерево в категории Стандарт (с распилом, погрузкой и вывозом). Финальная зависит от: высоты, диаметра ствола, доступа техники, удалённости от МКАД. Для удалённых районов (Сергиев Посад, Зарайск) — коэффициент +10-15%. Точную смету присылаем в течение 2 минут после загрузки фото в форму заявки.

Q 02Нужно ли согласование для спила?

На частном участке (ИЖС, СНТ) — согласование не требуется: дерево считается имуществом собственника. Исключение — деревья в природных охранных зонах (вы получаете уведомление от лесхоза при покупке участка).

На территориях МКД, городских парков, магистралей — нужен порубочный билет. Мы помогаем с оформлением (Москва — 7-21 день, МО — до 30 дней).

Q 03Какая гарантия на работы?

Гарантия зависит от tier'а:

  • Эконом — без гарантии (оставляем щепу на участке)
  • Стандарт — 14 дней на пень и уборку
  • Премиум — 30 дней + рекультивация газоном
Q 04Работаете зимой?

Да, круглый год. Зимой — даже удобнее: листва не мешает оценке, грунт промёрзший (техника не вязнет), цены такие же. Чистка крыш и спил аварийных деревьев — пик сезона декабрь-март.

Q 05Кто несёт ответственность при ЧП?

«Обиход» застрахован по ОПО на 5 млн ₽ (страховщик «РЕСО-Гарантия»). При повреждении вашего имущества или соседского — выплачиваем без судов. Бригадиры в СРО, страховка ОСАГО на технику обязательна.

Q 06Как происходит оплата?

B2C: аванс 30% после выезда оценщика, 70% — после акта приёмки. Картой, переводом, СБП. Чек ЕСИА.

B2B (УК / ТСЖ / застройщик): 100% по факту с УПД и счёт-фактурой. Постоплата до 14 дней. Договор подряда + ППР.

Q 07Куда вывозите ветки и древесину?

На лицензированный полигон (Росприроднадзор № 077-АБВ). Талон утилизации передаём вместе с актом — нужен B2B для отчётности и УК для жильцов. Качественную древесину можем продать на распил для бань / отопления — со скидкой к смете.

Q 08Можно заказать в выходные?

Да, без наценки за выходные. Аварийные выезды (упало дерево, угрожает крыше) — в течение 4-12 часов с момента заявки. Ночные работы — по согласованию + 30%.

33 · Header & Footer · canonical

Один Header, один Footer · iron rule единства chrome

На всём сайте один header (`site/components/marketing/Header.tsx`) и один footer (`site/components/marketing/Footer.tsx`). Не делаем shop-special-header, не добавляем суффиксы к лого, не урезаем nav для разных разделов. Header и footer — это site-chrome, единый каркас обёртки для всех 6 направлений (4 услуги + ландшафт + магазин) и для кабинета.

Iron rule: любая «своя версия» header/footer ломает доверие и навигацию. Если для конкретного экрана нужна минимальная chrome (auth-screens §31, thank-you-page §24, error-pages §11.5) — это deliberate UX exception, явно прописанная в этом разделе. Всё остальное использует canonical chrome из §33.

33.3 · Header · что обязательно одинаково везде

  • Логотип: wordmark «ОБИХОД» (без суффиксов вроде «· магазин» / «· ЛК» / «· admin»). Suffix допустим только в admin /admin (Payload), потому что admin — отдельный визуальный мир.
  • Телефон: один номер из SiteChrome-config (Payload). Меняется в одном месте — отображается везде.
  • Mega-menu пунктов: 6 направлений (Услуги / Районы / Магазин / Дизайн ландшафта / Кейсы / Цены) + 2 ссылки (Блог / Контакты). Состав не меняется по разделам.
  • Auth-блок справа: logged-out → CTA «Войти + Регистрация»; logged-in → user-menu (см. §30).
  • Sticky: header sticky на всех экранах desktop. Mobile — sticky тоже, но компактнее.
  • Skip-link «Перейти к содержимому» — обязательный (a11y).
  • Mobile breakpoint 860 px — burger-drawer вместо desktop-nav. Поведение идентично на всех страницах.
