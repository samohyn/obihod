# DistrictCard · spec

**Статус:** v0.9 · draft · роль `ui` · утверждено `art`
**Бизнес-контекст:** Обиход использует programmatic SEO — `<service> в <district>` страницы как основу органического трафика. 4 кластера услуг × 7 пилотных районов (Одинцово, Красногорск, Мытищи, Химки, Истра, Пушкино, Раменское) = 28 LP в первом seed + расширение до 60+ через 3 месяца. DistrictCard — карточка района на индекс-страницах `/uslugi/<cluster>/` и `/rajony/`.

## Purpose

Показать доступность услуги в конкретном районе МО за 1 секунду. Донести фикс-цену и местную специфику (ближайший кейс, сроки выезда, адаптация под район). Привести на programmatic LP района.

## Variants

| Variant | Когда |
|---|---|
| `default` | Сетка на `/rajony/` — все 28 районов + расширенные |
| `service-context` | На `/uslugi/arboristika/` — только карточки где эта услуга активна |
| `featured` | Выделенная карточка (hero-блок, priority-район — Раменское / Одинцово по сезону) |
| `compact` | Inline-лист в FAQ «Работаете в X?» — только название + галочка |

## States

| State | Border | Background | Shadow |
|---|---|---|---|
| `default` | 1 px `var(--c-line)` | `var(--c-card)` | none |
| `available` (услуги активны сегодня) | 1 px `var(--c-line)` | `var(--c-card)` | — |
| `high-demand` (сезонный пик) | 1 px `var(--c-accent)` | `var(--c-card)` | — · badge «Горячо» янтарный |
| `waitlist` (нет окон на неделю) | 1 px `var(--c-line)` | `var(--c-bg-alt)` | — · badge «Запись» muted |
| `hover` | 1 px `var(--c-primary)` | `var(--c-card)` | `{shadow.sm}`, `translateY(-2px)` |
| `focus-within` | 2 px `var(--c-primary)` | `var(--c-card)` | focus-ring |

## Token usage

- Padding: `{spacing.6}` = 24 px
- Border-radius: `{radius.xl}` = 16 px
- Gap между блоками внутри: 12-16 px
- Цвет якоря цены: `var(--c-primary)` + mono + tabular-nums

## Copy (TOV)

### Название района
Без приставок «г.», «п.», «ПГТ». Просто «Раменское», «Красногорск», «Одинцово». Если поселение внутри района (Быково, Удельная) — через дефис: «Раменское — Быково».

### Sub-caption
Факт + цифра, максимум 2 элемента:
- «от Москвы 47 км · 120 тыс жителей»
- «18 объектов закрыто в 2026»
- «ближайший выезд за 2 часа»

Не использовать: «крупный подмосковный город», «перспективный район» (оценочно, без ценности).

### Micro-promise
- **Do:** «Смета за 10 минут · выезд за 24 ч», «В работе 3 бригады», «Ближайший объект — сегодня 14:00»
- **Don't:** «Качественно и быстро», «Индивидуальный подход» (анти-TOV)

### Якорь цены
`от 12 800 ₽ / объект` — если амплитуда реальная. Фикс — `12 800 ₽ / объект` без «от».

### CTA
«Узнать подробнее» / «Посмотреть кейсы района» / «Оставить заявку в {{ district }}»

## A11y

- **Целиком link:** `<a href="/rajony/{slug}" class="district-card">` — ведёт на LP района.
- **Badge «Горячо» / «Запись»:** не только цвет, но и иконка + текст (для дальтоников, WCAG 1.4.1).
- **Mini-map-illustration:** декоративная → `aria-hidden="true"`. Не ставим в alt «карта района» — это не несёт полезной информации.
- **Список услуг:** `<ul role="list">` с семантической разметкой, не divs.
- **Touch-target:** карточка ≥ 200×260 px — много выше WCAG min 44.

## Specifics

### Mini-map illustration

**Опциональный элемент** — SVG-схема района с границей + M-маркером столицы МО относительно. Стиль чертежа (stroke-only, current-color). Не фотографическая карта, не google maps screenshot.

Если SVG нет для района — не показываем, карточка живёт без mini-map.

### Сезонная адаптация (для `service-context` варианта)

Зимой (декабрь-март) Крыши/снег выделяется — бейдж `{c.accent}` «В сезоне». Летом (июнь-август) — арбористика. Переключение через Payload `SiteChrome.seasonalHighlight` (TBD — backend-задача).

### SEO

Карточка содержит `<h3>` с названием района для `/uslugi/<cluster>/` индекса — помогает с programmatic ranking. На `/rajony/` — `<h2>`. Уровень heading определяется контекстом (prop `headingLevel`).

## Edge cases

- **Новый район без кейсов:** hide «18 объектов закрыто», показать «Бригада готова к первому выезду» + CTA «Быть первым».
- **Район с waitlist на неделю:** badge + адаптированный CTA «Записаться на следующую неделю» + captured дата.
- **Нет цены (услуга индивидуальная):** убираем якорь, ставим «Цена по объёму» body-sm italic.
- **Очень длинное название:** line-clamp: 2, не обрезаем hyphen'ом.
- **Русский + национальный район (Зарайск):** без транслитерации в URL, только кириллический slug — `/rajony/zarayск/` не работает, `/rajony/zarayskiy/` — стандарт.

## Integration

- **Данные:** Payload collection `Districts` + связь с `ServiceDistricts` (см. [US-1 seed](../../../specs/US-1-seed-prod-db/)).
- **Компонент:** Server Component в App Router — данные через Payload LocalAPI, без client-side fetch.
- **Кэширование:** `unstable_cache` на 1 час (списки услуг меняются редко).
